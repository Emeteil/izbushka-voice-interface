# База знаний СибГУТИ — неструктурированные документы

В этой папке лежат markdown-файлы со свободным текстом — то, что неудобно
складывать в YAML (`knowledge_base/sibsutis.yml`).

## Как этим пользуется ассистент

1. Сначала ассистент пытается найти ответ через `search_knowledge` (структурированная БЗ).
2. Если не находит — вызывает `list_documents`, видит список файлов по их именам и
   коротким аннотациям из заголовка первой строки.
3. По релевантным именам открывает нужный документ через `read_document(name)`.
4. Если ответа нет нигде — вызывает `log_missing_info(question, note)`, чтобы
   сотрудники приёмной потом увидели пробелы в базе и пополнили её.

## Соглашения

- Имя файла — латиницей в kebab-case, должно отражать тему. Пример:
  `military-department.md`, `accreditation.md`, `partner-companies.md`.
- Первая строка каждого файла — `# Заголовок темы` (используется как краткое описание
  при `list_documents`).
- Допускается любой markdown внутри: списки, таблицы, цитаты.
- Файлы, начинающиеся с `_`, считаются служебными и не индексируются.
