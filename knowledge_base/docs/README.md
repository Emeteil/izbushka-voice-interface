# База знаний — Руководство по заполнению

База знаний состоит из двух частей: **структурированной** (`sibsutis.yml`) и **неструктурированной** (markdown-файлы в папке `docs/`). Ассистент сначала ищет ответы в структурированной базе (`search_knowledge`), и только если не находит — обращается к неструктурированной (`list_documents` и `read_document`).

---

## 1. Структурированная база знаний (`sibsutis.yml`)

Файл `sibsutis.yml` содержит основные и часто запрашиваемые данные. Он загружается и парсится `loader.py`. 

### Основные разделы
- `university`: Общая информация о вузе (названия, город, год основания, сайт).
- `admission`: Информация о приемной кампании (контакты, сроки приема, документы, способы подачи). **Важно:** Этот раздел имеет приоритет (бонусный балл) в поиске.
- `faculties`: Список факультетов (`id`, `name`, `short`).
- `programs`: Образовательные программы. 
  - Обязательные поля: `id`, `title`, `faculty`, `degree`, `duration_years`, `forms`, `exams`, `description`.
  - **Критически важно:** Поле `keywords` (массив ключевых слов). Поиск по базе `_score_fragment` начисляет наибольший балл (3.0) при совпадении запроса с ключевыми словами программы.
- `scores`: Проходные баллы и количество мест с привязкой к `id` программ из раздела `programs`.
- `scholarship`: Размеры стипендий и примечания по их выплатам. Автоматически индексируется по словам "стипендия", "стипендии", "выплаты".
- `dormitory`: Информация об общежитии (стоимость, доступность). Автоматически индексируется по словам "общежитие", "общага", "жильё", "иногородним".
- `benefits`: Список льгот. При поиске автоматически триггерится на слова: "льгота", "льготы", "целевое", "олимпиада".
- `events`: Мероприятия (дни открытых дверей, олимпиады).
- `frequently_asked`: Список частых вопросов (поля `q` и `a`). **Важно:** Этот раздел также имеет повышенный приоритет (бонусный балл) при поиске.

### Правила заполнения YML
1. Избегайте лишней воды, пишите по факту. Текст собирается в поисковые фрагменты в `loader.py`.
2. Обязательно используйте `keywords` в направлениях (`programs`) для синонимов и сленговых названий специальностей, которые могут использовать абитуриенты (например, "иб", "программирование", "дизайн").
3. Проверяйте, что `id` факультетов и программ совпадают везде (например, ключи в `scores` должны точно соответствовать `id` в `programs`).

## 2. Неструктурированные документы (Папка `docs/`)

В этой папке лежат markdown-файлы со свободным текстом — то, что неудобно или невозможно жестко структурировать в YAML (например, подробные инструкции, приказы, специфические правила).

### Соглашения
- **Имя файла:** латиницей в kebab-case, должно отражать тему. Пример: `military-department.md`, `accreditation.md`, `partner-companies.md`.
- **Заголовок:** Первая строка каждого файла обязательно должна быть заголовком первого уровня (например, `# Военная кафедра`). Это используется как краткое описание (аннотация) при вызове `list_documents`.
- **Содержимое:** Допускается любой markdown внутри: списки, таблицы, цитаты. Главное — читаемость. Если текст больше 8000 символов, он будет обрезан при выдаче ассистенту.
- **Служебные файлы:** Файлы, начинающиеся с `_`, считаются служебными и не индексируются (как, например, этот файл `README.md`, который тоже игнорируется загрузчиком, так как его имя "readme.md").

### Как ассистент использует документы
1. Сначала ассистент обращается к `search_knowledge` для поиска по `sibsutis.yml`.
2. Если ответ не найден, он вызывает `list_documents`, получая список файлов (имена и заголовки).
3. Найдя подходящий файл, ассистент читает его через `read_document(name)`.
4. Если ответа нет ни в YML, ни в документах, ассистент вызывает `log_missing_info` (или `log_missing`), чтобы записать недостающую информацию в лог, и сотрудники могли позже дополнить базу знаний.
