# Izbushka Voice Interface

Мультимодальный голосовой ассистент для робототехнического комплекса Izbushka, реализованный на базе **Google Gemini 2.5 Live API** (Speech-to-Speech).

## Описание архитектуры

Интерфейс обеспечивает непрерывный двусторонний речевой обмен между пользователем и роботом. Для обработки контекста используется модель `gemini-2.5-flash-native-audio-preview`.

### Ключевые возможности

1.  **Мультимодальность**: Ассистент анализирует не только аудио-поток с микрофона, но и видео-поток с веб-камеры MJPEG из [izbushka-web-core](https://github.com/Emeteil/izbushka-web-core). Это позволяет модели Gemini визуально идентифицировать окружающую обстановку.
2.  **Tool Calling**: Реализована система выполнения произвольных функций на стороне клиента. Например, инструмент `SetEmotion` выполняет HTTP-вызовы в API [izbushka-web-core](https://github.com/Emeteil/izbushka-web-core) для динамического изменения эмоций робота.
3.  **Контекстная память**: Реализовано автоматическое сохранение выжимки диалогов в JSON для поддержания истории взаимодействий в будущих сессиях.
4.  **Фоновое распознавание (Wake Word)**: Детектирование слова-активатора с использованием [openWakeWord](https://github.com/dscripka/openWakeWord).

## Зависимости и сабмодули

Проект построен на основе следующих внешних компонентов:
*   [gemini_engine](https://github.com/Emeteil/gemini_engine) - базовый движок для регистрации инструментов и потоковой передачи данных в Gemini.
*   [wake_word](https://github.com/Emeteil/wake_word/) - оболочка для управления системой распознавания триггерных слов.

## Интеграция с Izbushka

Сервис работает в качестве клиента по отношению к [izbushka-web-core](https://github.com/Emeteil/izbushka-web-core) и требует для авторизации запросов `MASTER_TOKEN` и соответствующий `GEMINI_API_KEY`.
